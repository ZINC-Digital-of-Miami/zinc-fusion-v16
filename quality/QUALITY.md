## Purpose
Quality in ZINC-FUSION-V16 means buyer-facing soybean oil procurement decisions are grounded in accurate, traceable, and operationally consistent data contracts across Supabase, API routes, dashboard rendering, and training gates.

Deming applies here as contract integrity built into route code, schema contracts, and quality artifacts before later review phases. Juran applies as fitness for procurement use: the system must preserve fallback behavior, route-envelope stability, and approval-gated model operations so Chris and Kevin do not consume misleading outputs. Crosby applies because preventing silent contract drift (status vocab mismatch, hidden overlay disablement, envelope divergence) is lower cost than downstream outage analysis and procurement mis-timing.

## Coverage Theater Prevention
The following are non-compliant test patterns for this repository:

- Asserting only HTTP status without validating payload contract fields (`ok`, `data`, `asOf`, `source`) where envelope semantics are required.
- Asserting that forecast/target arrays are non-empty without proving horizon normalization and per-route parity behavior.
- Testing ingestion telemetry by checking row existence only, without validating canonical status vocabulary alignment across SQL and TS writers.
- Testing chart rendering by mounting the component only, without checking that forecast-target overlay attachment path is enabled when target-zone data exists.
- Testing training gates by asserting dry-run returns a dict, without asserting non-approved non-dry behavior is blocked.
- Treating README existence as documentation quality; tests must assert repo policy alignment on prohibited hardcoded port guidance.

## Fitness-to-Purpose Scenarios
### Scenario 1: Intraday Fallback Contract Drift
**Requirement tag:** [Req: code-derived — REQ-001]

**What happened:** `app/api/zl/intraday/route.ts` states a 15-minute then 1-minute fallback contract but currently executes only `mkt.price_15m` and always emits `source: "mkt.price_15m"`. During sparse 15-minute data windows, this permits empty/stale intraday payloads while a populated 1-minute table exists.

**The requirement:** Intraday route behavior must execute documented fallback semantics and emit source metadata that reflects the actual query path.

**How to verify:** `python -m unittest quality.test_functional.SpecRequirementTests.test_req_001_intraday_route_contains_executable_1m_fallback`

### Scenario 2: Invisible Target Zones Despite Available Data
**Requirement tag:** [Req: code-derived — REQ-002]

**What happened:** Dashboard page fetches `targetZones` and passes them to `ZlCandlestickChart`, but `SHOW_FORECAST_TARGET_OVERLAY = false` prevents `ForecastTargetsPrimitive` attachment. The architecture permits valid forecast-zone data to remain invisible to users.

**The requirement:** When target-zone data is supplied to chart boundary, overlay plumbing must be enabled or explicitly replaced by an equivalent visible projection.

**How to verify:** `python -m unittest quality.test_functional.FitnessScenarioTests.test_scenario_2_forecast_overlay_gate_is_not_hard_disabled`

### Scenario 3: Divergent Forecast Target API Envelopes
**Requirement tag:** [Req: code-derived — REQ-002]

**What happened:** `/api/zl/target-zones` uses a typed envelope while `/api/zl/forecast-targets` uses `{asOfDate, targets}` and route-specific error shape. This permits parser duplication and inconsistent error handling across consumers of equivalent domain data.

**The requirement:** Forecast-target routes must share one canonical response envelope or provide explicit typed adapters documenting differences.

**How to verify:** `python -m unittest quality.test_functional.SpecRequirementTests.test_req_002_forecast_target_routes_share_api_envelope_contract`

### Scenario 4: Ingest Status Vocabulary Fragmentation
**Requirement tag:** [Req: code-derived — REQ-003]

**What happened:** SQL jobs write lower-case `running/ok/failed` while TS helper writes upper-case `RUNNING/SUCCESS/FAILED` to `ops.ingest_run.status`. This permits split analytics filters and inconsistent alert logic.

**The requirement:** All writers to `ops.ingest_run.status` must use one canonical status vocabulary.

**How to verify:** `python -m unittest quality.test_functional.SpecRequirementTests.test_req_003_ingest_status_vocab_is_canonical_across_writers`

### Scenario 5: Unauthorized Model Execution
**Requirement tag:** [Req: code-derived — REQ-004]

**What happened:** Train and promote phases are safety-critical operations. If approval gates are bypassed in non-dry runs, model artifacts or cloud tables can mutate without explicit operator authorization.

**The requirement:** Non-dry `train` and `promote` execution requires explicit approval and emits a blocking error when approval is absent.

**How to verify:** `python -m unittest quality.test_functional.SpecRequirementTests.test_req_004_train_phase_requires_explicit_approval`

### Scenario 6: Runtime External Pull Coupling on Read Paths
**Requirement tag:** [Req: code-derived — REQ-006]

**What happened:** AI-first routes still perform request-time trusted-market pulls before snapshot fallback. Third-party latency/outage can cascade into user-facing API delays.

**The requirement:** Runtime read-path external pulls must either be bounded with explicit fallback semantics or isolated behind snapshot-first behavior with clear degraded metadata.

**How to verify:** `python -m unittest quality.test_functional.FitnessScenarioTests.test_scenario_6_routes_document_snapshot_and_trusted_pull_order`

### Scenario 7: Snapshot Trust Boundary Drift
**Requirement tag:** [Req: code-derived — REQ-004]

**What happened:** `TRUSTED_SNAPSHOT_SOURCES` defines a strict allowlist. If producers are added without allowlist updates, snapshots fail closed and AI cards silently lose expected data.

**The requirement:** Any new snapshot source identifier must be updated in the trust allowlist in the same change as producer wiring.

**How to verify:** `python -m unittest quality.test_functional.BoundaryAndEdgeCaseTests.test_boundary_trusted_snapshot_sources_are_exactly_expected_set`

### Scenario 8: Operator Guidance Drift
**Requirement tag:** [Req: code-derived — REQ-012]

**What happened:** Root README includes starter-template startup guidance including localhost:3000 while authority docs prohibit hardcoded port assumptions. This can mislead operators and parity validation steps.

**The requirement:** Operator-facing docs must not reintroduce prohibited startup assumptions that conflict with V16 authority rules.

**How to verify:** `python -m unittest quality.test_functional.FitnessScenarioTests.test_scenario_8_readme_drift_is_detectable`
