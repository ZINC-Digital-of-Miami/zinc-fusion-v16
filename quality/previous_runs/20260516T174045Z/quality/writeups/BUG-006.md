## Summary
`warning` optionality is inconsistent between API contract and parity gate in `lib/contracts/api.ts:8`; `scripts/verify/gate6.sh:79,99,120,138,154,170` causes this observable failure: gate requires `warning` key and string type on all checked endpoints.

## Spec reference
- Requirement: REQ-004
- Spec basis: REQ-004 condition 1 requires optional fields to remain optional across surfaces.
- Behavioral contract quote: "Optional fields are optional everywhere they appear."

## The code
At `lib/contracts/api.ts:8`; `scripts/verify/gate6.sh:79,99,120,138,154,170`, the current implementation behavior is: gate requires `warning` key and string type on all checked endpoints.

## Observable consequence
Expected behavior: gate checks must treat `warning` as optional when contract is `warning?: string`. Actual behavior: gate requires `warning` key and string type on all checked endpoints.
