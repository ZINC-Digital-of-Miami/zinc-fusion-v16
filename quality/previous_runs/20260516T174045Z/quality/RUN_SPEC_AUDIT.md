## The Definitive Audit Prompt

Run this exact prompt with three independent auditors.

---

**Context files to read:**
1. `quality/QUALITY.md`
2. `quality/REQUIREMENTS.md`
3. `quality/EXPLORATION.md`
4. `AGENTS.md`
5. `docs/plans/2026-03-17-v16-migration-plan.md`

**Task:** Act as the Tester. Compare implementation against requirements. Report only defects.

**Rules:**
- Every finding must include file:line evidence.
- Grep before claiming missing behavior.
- Read full function body before claiming behavior exists.
- Classify each finding as `MISSING`, `DIVERGENT`, `UNDOCUMENTED`, or `PHANTOM`.
- Use requirement tags in output (`REQ-001`..`REQ-006`).

**Project-specific scrutiny areas:**
1. Cross-writer `ops.ingest_run` lifecycle consistency (`run_id`, status vocabulary, terminal updates).
2. `/api/cron` assumptions in middleware and gate scripts vs architecture policy.
3. `/api/zl/intraday` fallback promise vs executed query paths.
4. Envelope semantics between `lib/contracts/api.ts`, route handlers, and `gate6.sh`.
5. Risk-factor freshness semantics and mixed-vintage messaging continuity API→UI.
6. Horizon normalization parity across forecast, target-zones, and forecast-targets routes.
7. Snapshot whitelist rejection behavior and fallback observability.

**Output format:**

  Spec says: ...

## Confidence and Triage Rules

| Confidence | Found By | Action |
|---|---|---|
| Highest | 3/3 | Confirm unless verification probe disproves |
| High | 2/3 | Confirm with targeted verification |
| Needs verification | 1/3 | Run verification probe before classification |

## Verification Probes Must Be Executable

For disputed findings, create executable assertions in one of:

- `quality/mechanical/verify.sh`, or
- `quality/spec_audits/triage_probes.sh`

Requirements:
- Rejections require passing assertions proving claim false.
- Confirmations require failing assertions proving claim true.
- Every assertion cites exact evidence line.

## Categorize Confirmed Findings

Use one of:
- Spec bug
- Design decision
- Real code bug
- Documentation gap
- Missing test
- Inferred requirement wrong

## Cross-Artifact Consistency

Compare triage against Phase 3 code review findings.
If conflict exists on same factual claim, run verification probe and record resolution.

## Fix Execution Rules

- Group fixes by subsystem.
- Avoid mega-prompts.
- Require regression tests for real code bugs.
