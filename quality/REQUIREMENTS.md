## Requirements
### REQ-001: Intraday fallback execution parity
- Statement: `app/api/zl/intraday` must execute the declared 15m->1m fallback behavior when primary query returns no usable bars.
- Tier: 3
- Source type: code-derived
- Pattern: parity
- References:
  - `app/api/zl/intraday/route.ts:16-23`
  - `supabase/migrations/202603180002_mkt.sql:18-19`
- Linked use cases: `UC-1`
- Conditions of satisfaction:
  - `GET()` includes an executable branch that queries `mkt.price_1m` when `mkt.price_15m` produces no usable rows.
  - Response `source` reflects the table actually used for the payload.
  - Fallback behavior is mechanically testable by source-level contract assertions in `quality/test_functional.py`.

### UC-1: Serve intraday bars with executable fallback
- Actors: dashboard chart consumer, `/api/zl/intraday` route
- Preconditions: 15-minute table can be sparse while 1-minute table contains bars
- Flow:
  - Query `mkt.price_15m`
  - If unusable/empty, query `mkt.price_1m`
  - Return normalized envelope with accurate `source`
- Postconditions: route behavior matches declared fallback contract
- Requirement links: `REQ-001`
