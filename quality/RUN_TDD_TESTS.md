# TDD Verification Protocol: ZINC-FUSION-V16
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-05-16 · Project: ZINC-FUSION-V16

## Purpose
Validate every confirmed bug with a red/green cycle and produce patch-ready evidence without mutating source outside `quality/`.

## Inputs
- `quality/BUGS.md`
- `quality/patches/BUG-<id>-regression-test.patch`
- Optional `quality/patches/BUG-<id>-fix.patch`
- Project test command(s)

## Procedure Per Bug
1. Red phase:
   - Apply regression-test patch in disposable worktree.
   - Disable skip/xfail guard for this run.
   - Run targeted regression test.
   - Expect failure.
2. Green phase:
   - Apply fix patch (if present) plus regression-test patch.
   - Re-run targeted regression test.
   - Expect pass.
3. Record outcomes in sidecar JSON.

## Output Files
- `quality/results/tdd-results.json` (`schema_version: "1.1"`, `skill_version: "1.5.7"`)
- `quality/results/recheck-results.json` (`schema_version: "1.0"`)
- Bug writeups in `quality/writeups/`

## Required Sidecar Fields
`tdd-results.json` must include:
- `schema_version`, `skill_version`, `date`, `project`
- `bugs_total`, `bugs_verified`, `bugs_open`
- `results[]` with `bug_id`, `red_phase`, `green_phase`, `verdict`, `regression_test`, `fix_patch`, `writeup_path`

## Verdict Rules
- `TDD verified`: red fails, green passes
- `confirmed open`: red fails, green skipped (no fix patch)
- `needs investigation`: red passes unexpectedly or test misalignment detected

## Guardrails
- Do not apply patches to working source tree directly.
- Use disposable worktree for destructive apply/run steps.
- Keep skip/xfail guards restored after verification runs.
