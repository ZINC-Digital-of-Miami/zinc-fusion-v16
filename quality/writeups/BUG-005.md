## Summary
`components/chart/ZlCandlestickChart.tsx` hard-disables the forecast target overlay gate, preventing target-zone projection from rendering even when target-zone data is present.

## Spec reference
- Requirement: REQ-002
- Spec basis: REQ-002 contract mapping (`C-006`, `C-007`) requires target-zone signal visibility when target-zone data is provided.
- Behavioral contract quote: "target-zone signal visibility when target-zone data is provided"

## The code
At `components/chart/ZlCandlestickChart.tsx:40`, `SHOW_FORECAST_TARGET_OVERLAY` is set to `false`, and the attach/render branches at `components/chart/ZlCandlestickChart.tsx:260-269` therefore never execute.

## Observable consequence
Reading the chart module shows the overlay gate is statically disabled, so target-zone overlay behavior is unreachable at runtime. Expected behavior is visible overlay projection when `targetZones` are supplied; actual behavior is a permanently disabled overlay path.
