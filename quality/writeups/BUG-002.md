## Summary
`app/api/zl/forecast-targets/route.ts` returns a divergent payload shape (`asOfDate`, `targets`) instead of the canonical envelope keys used by the sibling target-zones route.

## Spec reference
- Requirement: REQ-002
- Spec basis: REQ-002 requires equivalent forecast-target endpoints to expose one stable envelope contract (or explicit typed adapters).
- Behavioral contract quote: "one stable envelope contract (or explicit typed adapters)"

## The code
At `app/api/zl/target-zones/route.ts:80-85`, the sibling route emits a canonical envelope, while `app/api/zl/forecast-targets/route.ts:103-106` emits route-specific keys and does not construct the same envelope contract.

## Observable consequence
Comparing both route handlers reproduces the divergence immediately: `/api/zl/forecast-targets` does not emit canonical `ok`, `data`, and `asOf` keys. Expected behavior is one stable envelope across equivalent payloads; actual behavior is consumer-visible contract split.
