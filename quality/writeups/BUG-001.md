## Summary
`GET()` in `app/api/zl/intraday/route.ts` never executes the documented 15m->1m fallback path, so the route cannot return `price_1m` bars when 15-minute bars are unavailable.

## Spec reference
- Requirement: REQ-001
- Spec basis: REQ-001 requires executable 15m->1m fallback and source labeling aligned to the active table.
- Behavioral contract quote: "executable 15m->1m fallback and source labeling aligned to the active table"

## The code
At `app/api/zl/intraday/route.ts:16-23` and `app/api/zl/intraday/route.ts:41-46`, the handler only queries `price_15m` and emits a fixed `source` label, with no fallback branch to `price_1m`.

## Observable consequence
When `mkt.price_15m` has no usable rows, reading `app/api/zl/intraday/route.ts` shows no `.from("price_1m")` path, so fallback never executes. The expected behavior is to query `mkt.price_1m` and report the active source table; the actual behavior is that only `mkt.price_15m` is queried and reported.
