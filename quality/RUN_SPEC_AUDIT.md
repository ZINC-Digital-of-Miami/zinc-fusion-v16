# Spec Audit Protocol: ZINC-FUSION-V16
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-05-16 · Project: ZINC-FUSION-V16

## Council Setup
Run three independent auditors with no shared findings until triage:
- Auditor A: requirement-first strict contract verification
- Auditor B: architecture consistency and cross-file behavior verification
- Auditor C: adversarial boundary and omission-focused verification

## Inputs
1. `quality/REQUIREMENTS.md`
2. `quality/QUALITY.md`
3. `quality/CONTRACTS.md`
4. `quality/COVERAGE_MATRIX.md`
5. Relevant source files cited by each REQ

## Output Locations
- Auditor reports: `quality/spec_audits/YYYY-MM-DD-auditor-1.md` .. `-3.md`
- Triage synthesis: `quality/spec_audits/YYYY-MM-DD-triage.md`
- Probe script: `quality/spec_audits/triage_probes.sh`
- Probe output capture: `quality/spec_audits/triage_probes.out.txt`

## Audit Procedure
1. Each auditor reviews all REQs independently and records per-REQ verdicts with code citations.
2. Triage compares overlaps, disagreements, and net-new findings.
3. Every confirmation or rejection using a probe must be executable evidence:
   - Write assertions in `triage_probes.sh` or `quality/mechanical/verify.sh`.
   - Execute probes and capture output file.
4. Classify each finding as:
   - `Real code bug`
   - `Spec gap`
   - `False positive`
   - `Needs follow-up`

## Mandatory Finding Fields
Each triage finding includes:
- `id` (`BUG-NNN` if confirmed)
- `requirement` (`REQ-NNN`)
- `source` (`code-spec` | `internal-prose` | `cross-source`)
- `severity` (`HIGH` | `MEDIUM` | `LOW`)
- `evidence` (file:line and/or probe output)
- `disposition` candidate

## Post-Triage Hooks
- Confirmed code bugs feed Phase 5 regression test workflow.
- Confirmed spec gaps feed requirement/spec update queue.
- Overturned code-review findings must be reconciled with regression artifacts.
